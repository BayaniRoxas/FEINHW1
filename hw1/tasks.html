Homework 1: Tasks for jr479

Your personal task is to develop a system for universities and their researchers.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Researcher. With the following private fields:

  1. String name
  2. String group
  3. String area
  4. int grade
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.University. With the following private fields:

  1. int id
  2. String chancellor
  3. String campus
  4. List<Researcher> researchers
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<University> universities in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one university with two researchers. [4 marks]

Create controller co2123.hw1.controller.UniversityController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/universities" should show view universities/list. [2 marks]
    • Add the list of all universities from co2123.hw1.Hw1Application to the model as attribute universities. [2 marks]
    • Create the corresponding JSP to list the chancellor and campus of each university followed by a link to URL "/researchers?university={id of university}", e.g., "/researchers?university=3". [4 marks]
  2. Requests for URL "/newUniversity" should show view universities/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="university" and action="/addUniversity" and inputs for all fields of class University except for researchers. [5 marks]
  3. POST requests for URL "/addUniversity" should add a new university to the list of all universities in co2123.hw1.Hw1Application and redirect to /universities. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.ResearcherController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/researchers" should show view researchers/list. [2 marks]
    • Based on request parameter university determine the corresponding university from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all researchers of that university to the model as attribute researchers. [2 marks]
    • Create the corresponding JSP for the view to list the name, group, area, and grade of each researcher. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newResearcher?university={id of university whose researchers are listed}", e.g., "/newResearcher?university=3". [4 marks]
  2. Requests for URL "/newResearcher" should show view researchers/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="researcher" and action="/addResearcher" and inputs for all fields of class Researcher. [6 marks]
    • Also add the request parameter university with the ID of the current university. [4 marks]
  3. POST requests for URL "/addResearcher" should add a new researcher to the list of researchers of the university identified by request parameter university and then redirect to /universities. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class University and make it available in controller UniversityController. [4 marks]

When adding a new university validate that

  1. The value of field id is unique within the list Hw1Application.universities. [3 marks]
  2. The value of field chancellor is not empty. [2 marks]
  3. The value of field campus is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Researcher and make it available in controller ResearcherController. [4 marks]

When adding a new researcher validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field group is not empty. [2 marks]
  3. The value of field area is either "Human Computer Interaction", "Machine Learning", or "Theory". [3 marks]
  4. The value of field grade is between (including) 7 and 10. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
